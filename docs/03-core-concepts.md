# 03. Core Concepts

## 五大核心能力

### 1) Control（控制）

定义输入输出边界、工具白名单、策略限制，避免 Agent 任意扩散。

### 2) Testing（测试）

将典型输入样例固化为测试集，验证输出结构、语义与业务规则。

### 3) Evaluation（评估）

通过准确率、一致性、时延、成本等指标量化系统表现。

### 4) Reproducibility（可复现）

固定版本、参数和数据，确保实验结果可重复、可对比。

### 5) Iteration（可演进）

基于评估反馈持续迭代 Prompt、工具调用和流程策略。

## 一句话总结

Harness Engineering 不是“让模型更聪明”，而是“让系统更可靠”。
